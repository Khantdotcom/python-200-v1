# Week 8: Introduction to Cloud Computing

Welcome to Week 8 of Python 200, and welcome to the cloud.

Over the past seven weeks you have built a strong foundation: statistical thinking, classical machine learning, applied ML patterns, large language models, RAG, and AI agents. That is a serious achievement, and you should feel proud. For the remaining four weeks, we will be moving some of your expertise online, to the cloud.

Cloud platforms are the infrastructure that makes modern data engineering possible — on-demand compute, scalable storage, managed databases, ML services, and the pipelines that tie it all together. In the final stretch of this course, you will learn to work in that environment. For the hands-on work, we will be using Supabase, a hosted PostgreSQL platform that is self-provisioned, free for course use, and a natural fit for the ETL pipeline you will build in weeks 9–11.

This first week is about getting situated conceptually. By the end of it, you will understand the core concepts of cloud computing, be able to orient yourself in the cloud provider landscape, and know what to expect from the Supabase-based weeks ahead.

## Topics

1. [Cloud Overview](./01_cloud_overview.md)
A platform-agnostic introduction to cloud computing: what it is, why it matters, and what changes when you move from running code on your laptop to running it in the cloud. Covers the core concepts — IaaS, PaaS, SaaS, managed services, scaling models — that you will encounter regardless of which cloud provider you use.

2. [The Cloud Provider Landscape](./02_cloud_landscapes.md)
A survey of the cloud provider ecosystem: the big three hyperscalers (AWS, GCP, Azure), developer-tier platforms (DigitalOcean, Render, Cloudflare), and backend-as-a-service platforms (Firebase, Supabase). Includes a service taxonomy that maps common data engineering categories — compute, object storage, managed databases, LLM APIs, ML platforms — across providers, and closes with an explanation of why this course uses Supabase for weeks 9–11.
